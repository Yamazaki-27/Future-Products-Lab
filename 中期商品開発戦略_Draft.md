# 株式会社スギヤス 中期商品開発戦略（ドラフト）

> 作成：2026年6月　技術部
> 出典：MODEX2026（アトランタ）、欧州出張（MANUVIT／IMS）、Computex2026（台北）、Interop Tokyo 2026（幕張）の各視察報告より統合

---

## 1. 現状認識（視察から得られた共通トレンド）

各出張報告を横串で見ると、以下の構造変化が同時に起きている。

| トレンド | 根拠（出典） |
|---|---|
| ① AMR/AGVの自律制御が「地図＋LiDAR」から「カメラ＋Vision AI（SLAM）」へ移行しつつある | MODEX（AMRが人混みを自律走行）／Computex oToBrite「LiDARより高精度」／Inventec iUGV（NVIDIA Jetson Orin NX + Isaac/cuVSLAM） |
| ② NVIDIA Jetsonがエッジロボットのデファクト標準になっている | MODEX・Computex・Interop（roboflow, Hailo-8）で共通言及 |
| ③ 現場の「見えないもの」（感情・状態・作業手順）をAIで可視化する流れ | Olive（感情可視化）、SORABITO TakumiX（音声AI）、NVIDIA Visual AI（SOP逸脱検知） |
| ④ 点検・保守のデジタル化＋行政対応が商機になっている | SORABITO（フォークリフト年次点検のWeb化・行政認可取得） |
| ⑤ データセンター物理層（リフト・冷却）が地味な高成長ニッチ | Interop ALPSystems serverLIFT／橋本GM発案のDC専用リフト構想 |
| ⑥ 欧州OEMとの連携余地（改造比率6割、3DCAD設計力） | MANUVIT／IMS訪問（9月物流展向けOEM供給検討） |
| ⑦ ソフト・AI起点の開発プロセス（仕様駆動開発）への転換 | Interop AWSセッション（AI-Driven開発／ポータブルスペック） |

**結論**：当社の主戦場（リフト・フォークリフト・AGV/AMR）は、ハード単体の性能競争から、**「センシング（カメラ/AI）」「点検・保守のデータ化」「現場ハードへのAI実装」**という3つの軸が重なる領域に移りつつある。中期戦略は、この重なりに当社の現場対応力を掛け合わせることを軸とする。

---

## 2. 中期戦略の3本柱（FY2026〜FY2029想定）

### 柱1：次世代AMR/AGV ─ 「脱LiDAR・カメラ主軸」への転換

- **方向性**：ステレオカメラ＋Vision AI（oToBrite型）とNVIDIA Jetson/Isaacプラットフォームを軸に、現行AGV/AMRの後継機を設計。LiDAR前提を一度外し、コストダウンと「カメラだけ」という訴求軸を商品コンセプトに据える。
- **短期（〜1年）**：
  - 現在進行中のIDEC・DMPとのコラボに、Inventec（iUGV／NVIDIA Orin NX・CANBus・RS485・LFP）をベンチマークとして接続。来週予定のメールコンタクトを足がかりに提携可能性を検証。
  - 自社AMR向けパレット認識を、roboflowを使った画像認識パイプラインで試作（水野が着手しているヒト検出カメラ研究と統合）。
- **中期（1〜3年）**：
  - 「地図ハイブリッド構成」（Inventecで確認済みの現実解）を基本アーキテクチャとして採用し、完全マップレス制御は研究テーマとして並走。
  - 欧州MANUVIT／IMSとのOEM提携を、9月の物流展出展に向けて具体化（橋本GMの見積依頼が起点）。AGV/AMRのハンド・KGL系商品を国内向けにローカライズ。

### 柱2：スマートリフト／フォークリフト ─ 点検・状態センシングの製品化

- **方向性**：リフト・フォークリフトという当社の本業に、「点検のデジタル化」「現場の状態（人・荷・機械）の可視化」を組み込み、ハード＋点検SaaS／センシングのセット提案に転換する。
- **具体施策**：
  - **点検デジタル化提携**：SORABITO（TakumiX／フォークリフト年次点検の行政認可済みWebシステム）との協業を本格検討。EP・トラバー電動走行タイプを対象に、点検記録のデジタル化をリフト本体の標準オプション化。
  - **状態・感情センシング**：Olive（静脈検出による感情リアルタイム可視化、トヨタ乗り心地開発が起源）の技術をリフト・製造ラインの操作者モニタリングに応用検討。安全性・生産性のKPI化につなげる。
  - **品質記録の信頼性保証**：DMP×STARBITの「AI動画改ざん防止エンジン」を、リフトの安全検査記録・工場内監視映像の証跡管理に展開できないか調査。
  - **データセンター専用リフト（ServerLift型）**：ALPSystems serverLIFTを参考に、クリーン度・静電気・耐荷重要件をクリアした専用機を新規開発。スーパーニッチでのNo.1ポジションを狙う中期の柱として明確に予算化する。

### 柱3：画像認識AI基盤＋現場ハードへの実装力の内製化

- **方向性**：roboflow等のクラウド型Vision AI基盤を使い、パレット・人物・設備状態の認識モデルを社内で継続的に作れる体制を作る。Hailo-8等のエッジAIチップ搭載「タフなPC」を組み合わせ、現場で壊れずに動くAI実装をセットで設計する。
- **具体施策**：
  - システムズナカシマ（roboflow担当・滝澤氏）とのWeb会議を起点に、パレット傾き検知モデルのPoCを実施。
  - 水野の「ヒト検出カメラ装置」研究（Jetson AGX Orin系）を、AMR・リフト・福祉機器に横展開できる共通モジュールとして標準化。
  - ARCRA「図面見積算出AI」など、CAD関連業務へのAI適用も技術調査の対象とする（柱4と連動）。

## 3. 横串：開発プロセス変革（柱0として全体を支える）

- Interopで得た「仕様駆動開発（Specification-Driven Development）」の考え方を、社内のCAD不具合管理システム・新商品開発プロセスに適用。
- 「仕様を先に言語化する」ことをGitHub運用の中心目的とし、AIエージェント（設計・実装・検証）を活用した開発スピードアップを、上記3本柱すべてのPoCに適用する。
- 既知パターンを「ポータブルスペック」として社内に蓄積し、属人化を防止。

## 5. KPIイメージ（ドラフト）

- 柱1：脱LiDAR試作AMRのPoC完了（1年以内）／欧州OEM商品の国内出展（9月）
- 柱2：フォークリフト点検デジタル化オプションのリリース（1.5年以内）／DC向けリフト試作（2〜3年）
- 柱3：パレット認識モデルPoC完了（半年以内）／ヒト検出モジュールの横展開3製品適用（2年以内）
- 横串：CAD不具合管理システムへのAI駆動開発適用、仕様書テンプレート化（1年以内）

## 6. 留意点・リスク

- 技術陳腐化のスピードが非常に速い（Computexで「犬型配膳ロボットはもはや誰も見ない」との指摘）。柱2・3の「現場×AI」「点検データ化」は、当社の現場対応力が活かせる差別化領域として優先度を高く維持する。
- データセンターリフトは要求スペック（クリーン度・静電気・耐荷重）が厳しく、開発リードタイムが長い前提で中期計画に組み込む。
- 提携検討先（Inventec, SORABITO, Olive, DMP等）は、いずれも展示会での名刺交換ベースのため、早期にNDA・Web会議設定まで進めることが重要。
