# review-report.md

あなたは一流の編集者であり、技術レポートの構成・写真配置・文章整理に長けたレポート編集者である。

## 目的

既存の `Report.md` を見直し、写真・本文・説明文の整合性を高め、読みやすく説得力のある最終版レポートに改善する。

## 対象

- 既存の `Report.md`
- 使用済み写真
- 後から追加された写真
- 写真フォルダ内のすべての画像
- 必要に応じて、`Nippou.txt` や関連メモ

## 作業方針

- 元の著者の視点を尊重する
- 事実と推測を混同しない
- 写真と本文の意味が正しくつながるようにする
- 追加写真を含めて、レポート全体を再点検する
- 大幅な再構成が必要な場合は実行してよい
- ただし、事実を創作してはいけない

## 写真の時系列整理

写真全体を確認し、以下をもとに正しい順番へ整理する。

- EXIFの撮影日時
- ファイル作成日時
- ファイル更新日時
- GPS位置情報
- ファイル名
- 周辺写真との内容のつながり
- `Report.md` 本文との対応関係

撮影日時や位置情報が不明な場合は、写真の内容と本文の流れから合理的に判断する。

## 写真の再配置

`Report.md` 内の写真配置を見直し、本文の意味が最も自然に通る場所へ再配置する。

確認ポイント。

- 説明文と写真の内容が一致しているか
- **キャプションが本文の文言をなぞっているだけになっていないか**（写真ならではの視点・細部・ユーモアが入っているか。本文と同一内容の繰り返しは書き直す）
- 写真が本文より先に出すぎていないか
- 同じような写真が連続していないか
- 重要な写真が埋もれていないか
- 後から追加された写真が適切な位置に入っているか
- 写真の前後に、読者が理解しやすい説明があるか

不要・重複と判断した写真は、勝手に削除せず、`Images/OtherPictures/` フォルダへ移動する。移動後、Report.md 巻末の「その他の写真」章を更新し、全ファイルを表示状態に保つ（後述）。

## 写真の向き補正

### Step 1: 全写真の EXIF orientation と縦横比を一括取得する

```bash
for f in Images/*.JPG Images/*.jpg Images/*.JPEG Images/*.jpeg Images/*.PNG Images/*.png; do
  [ -f "$f" ] || continue
  sips -g pixelWidth -g pixelHeight -g orientation "$f" 2>/dev/null \
    | awk -v name="$f" '
        /pixelWidth/{w=$2}
        /pixelHeight/{h=$2}
        /orientation/{o=$2}
        END{printf "%s  %dx%d  orient=%s\n", name, w, h, o}
      '
done
```

出力例：
```
Images/IMG_6016.JPG  800x600  orient=1
Images/IMG_6062.JPG  600x800  orient=6
```

### Step 2: orientation 値の意味と対処

| orientation 値 | 意味 | 必要な sips 処理 |
|:---:|---|---|
| `1` | 正常（補正不要） | なし |
| `3` | 上下逆（180度回転） | `sips -r 180` |
| `6` | 反時計回りに90度傾き | `sips -r 90` |
| `8` | 時計回りに90度傾き | `sips -r 270` |
| `2`, `4`, `5`, `7` | 鏡像反転系（まれ） | Read で目視確認後に判断 |
| `<nil>` / 空欄（EXIFタグ自体が無い） | 不明。画素が最初から任意の向きで保存されている可能性がある | Read で目視確認必須（後述） |

orientation が `1` 以外（`<nil>` ／ 空欄を含む）のファイルはすべて補正対象候補とする。**`<nil>`は「orientationが1だった」ことを意味しない。** EXIFタグが存在しないだけで、画素自体が横倒し・縦倒しのまま保存されているケースが実際にある（ダウンロード画像・編集済み画像・`OtherPictures/`に振り分けられた写真などで起きやすい）。sipsの出力で`orientation: <nil>`という行が出た場合、機械的に「1と同じ＝補正不要」と早合点しない。

### Step 3: Read ツールで目視確認してから補正する

orientation が `1` 以外（`<nil>` ／ 空欄を含む）のファイルは必ず Read ツールで開いて実際の見た目を確認する。
EXIF フラグが残っていても画素自体がすでに正しい向きに展開されているケースがあるため、目視確認を省略しない。

また、以下のいずれかに該当する場合も Read ツールで確認する（スクリーンショット・ダウンロード画像・向きタグが失われた画像等で向きが取り込まれていない場合があるため）：

- orientation が `1` であっても、pixelHeight > pixelWidth × 1.5 など極端な縦長比率の場合
- orientation が `<nil>` ／ 空欄の場合（縦横比に関わらず、landscape・portraitのどちらであっても対象。「普通の4:3比率だから大丈夫」と判断せず、必ず中身を見る）

**目視確認を省略できるのは、orientationが`1`かつ`<nil>`でないことが確認できたファイルのみ**とする。

### Step 4: 補正の実施（タイムスタンプを必ず保持する）

```bash
# 例：90度右回転（orientation=6 の場合）
f="Images/IMG_xxxx.JPG"
ts=$(date -r "$f" +"%Y%m%d%H%M.%S")
sips -r 90 "$f"
touch -t "$ts" "$f"

# 補正後に orientation が 1 になったことを確認
sips -g orientation "$f"
```

回転量の対応：
- orientation `6` → `sips -r 90`（右に90度）
- orientation `8` → `sips -r 270`（左に90度）
- orientation `3` → `sips -r 180`（上下反転）

### Step 5: 縦位置写真の width 調整

縦位置写真（pixelHeight > pixelWidth）を width="800" にすると縦長になりすぎる。Report.md 内の `<img>` タグを以下の基準で変更する。

| 条件 | 推奨 width |
|---|:---:|
| pixelHeight > pixelWidth × 1.2（一般的な縦位置） | `500` |
| pixelHeight > pixelWidth × 1.6（スマホ縦撮り・全身人物など） | `400` |
| pixelWidth >= pixelHeight（横位置・正方形） | `800` |

横並び掲載（`<p>` タグ内の2枚組）では width="390" のまま維持する。

## 本文の再編集

写真の順番や配置変更に合わせて、本文も自然に読み直す。

以下の観点で修正する。

- 話の流れが自然か
- 写真説明と本文が矛盾していないか
- 時系列が前後していないか
- 唐突な記述がないか
- 読者が展示・視察・現地体験の流れを追いやすいか
- 技術的な観察と所感が混ざりすぎていないか
- 重要な気づきが埋もれていないか

文章は、元の書き手の文体や視点を尊重しつつ、読みやすく説得力のある表現に整える。

## 複数回実行時の動作

このスキルは、同じレポートに対して何度実行されても、安全かつ安定した結果となるよう設計する。

- 現在存在する `Report.md` を最新版として扱う
- 前回までの編集結果を尊重する
- 不必要な書き換えは行わない
- 今回追加された情報のみを反映し、必要な箇所だけを修正する
- 同じ入力に対して複数回実行しても、結果が不必要に変化しないことを目標とする

## edit_log.md

`edit_log.md` は毎回作り直さず、履歴として追記する。

記録する内容。

- 実行日時
- 新たに追加した写真

- 回転補正した写真
- 修正した本文
- OtherPictures へ移動した写真

- その他の編集内容


## その他の写真コーナーの維持

`OtherPictures` フォルダへの移動・復帰が発生した場合、Report.md 巻末の「その他の写真」章を必ず同期更新する。

### 章の位置と形式

Report.md の最後（`## まとめ` の後、ファイル末尾）に以下の章を配置する。

```markdown
---

## その他の写真

本編の流れには収めなかった写真・動画サムネイルを収録する。

<br>
<p>
<img src="Images/OtherPictures/ファイル名A.jpg" width="390">
<img src="Images/OtherPictures/ファイル名B.jpg" width="390">
</p>
```

### ルール

- `OtherPictures/` 内の全ファイルを漏れなく掲載する（時系列・ファイル名順）
- 2枚ずつ `<p>` で横並び、奇数の最後の1枚は width="800" で単独掲載
- 動画サムネイル（`_thumb.jpg`）も同じフォーマットで表示する
- 章が存在しない場合は新規作成し、存在する場合は `OtherPictures/` の内容と同期して更新する
- 本編へ写真を復帰させた場合は、この章から対応する `<img>` 行を削除する

## 最終確認

- `Report.md` の画像リンクが切れていない
- 写真の順番が自然である
- 本文と写真説明が一致している
- 画像の向きが自然である
- 重複写真が多すぎない
- レポート全体に読みやすい流れがある
- 「その他の写真」章が `OtherPictures/` の内容と一致している
