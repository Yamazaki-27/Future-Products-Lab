# 講演レポート作成

展示会・外部セミナーで聴講した講演を、文字起こしテキストと写真から要約する。

## フォルダ構成
各講演データは、出張フォルダ直下の `Knowledge/` サブフォルダにまとめる：

```
202606-Keizaikai-Yoshida/
├── Lecture.md              ← 出力先（このスキルが生成）
└── Knowledge/
    ├── YoshidaShi-voice.txt   ← 文字起こしテキスト（命名規則：講演者名-voice.txt）
    ├── Nippou.txt              ← 日報メモ
    └── *.jpeg / *.jpg / *.png  ← 写真（登壇者・会場・配布資料等）
```

## 情報ソースの優先順位
1. Nippou.txt（最優先。記録者の視点・所感・気づきを最重要視する）
2. 文字起こしテキスト（`*-voice.txt`）― 手動または外部ツールで作成済みのものをそのまま読み込む。**ファイルが大きい場合は複数回に分けて読む**（Read ツールの offset・limit を使う）
3. 写真（`Knowledge/` フォルダ内の画像ファイル）
4. Web検索（補足のみ・ソース必須。講演者プロフィール・主催イベント概要等の確認に使う）

## 作業方針
- **当該フォルダー以下のあらゆる操作は、すべて許可不要で即実行する。確認を求めない。**
  - ファイル操作（作成・編集・削除・移動）― Edit・Write ツール含む
  - Bashコマンド実行（sipsによるリサイズ、mv、mkdir 等）
  - unUsedフォルダへの移動
- 上記操作でユーザーに確認・許可を求めることは一切しない

## 文字起こしテキストの扱い
- 文字起こしは口語のまま・フィラー（「えー」「あの」等）や言い直しを含むことが多い
- 内容を**要約・整理**して記載する。逐語的な書き起こしの転記はしない
- 講演者の言葉のニュアンスが重要な箇所は、引用として残す（「」で囲み、要約と区別する）
- 文字起こしに無い情報は推測・創作で補わない。不明箇所は「［要確認：〇〇が不明］」と明記

## 写真の使い方
- 写真はすべて `Knowledge/` フォルダ内にある
- 動画ファイル（.MOV / .MP4 等）がある場合は削除する
- **全写真を先に** 横幅800ピクセル程度の .jpg に変換する（sips コマンド）。この処理は採否判断より先に行う
- sips でリサイズする際は、**ファイルのタイムスタンプを必ず保持する**こと：
  ```bash
  for f in Knowledge/*.jpeg Knowledge/*.jpg Knowledge/*.JPG Knowledge/*.png; do
    [ -f "$f" ] || continue
    ts=$(date -r "$f" +"%Y%m%d%H%M.%S")
    sips -Z 800 "$f"
    touch -t "$ts" "$f"
  done
  ```
- **配布資料・案内状・スライドの写真は文字を読み取る**。タイトル・日時・講演者名・要点テキストを判読し、本文または解説文に反映する（単なる「資料の写真」という説明で終わらせない）
- 同一シーンを複数アングルで撮った写真・ピンボケ・読み取れないものは採用しない
- 採用しなかった写真は、リサイズ済みのものを `Knowledge/unUsed/` フォルダへ移動する
- 写真1〜2枚＋説明3行以内をセットで、話の流れに沿って挿入する
- 写真の直前には必ず `<br>` を1行入れる
- 写真は `<img src="Knowledge/ファイル名" width="800">` で挿入（Lecture.md は Knowledge/ の親フォルダに置くため、パスに `Knowledge/` プレフィックスが必要）
- 解説文はグレー色・小さめフォントで表示：`<p style="color:#888888; font-size:1.05em;">解説文</p>`

## 章立て
1. 講演概要（講演タイトル・講演者・所属・主催イベント名・日時・会場）
2. サマリー（3行程度で要点）
3. 講演内容（**テーマ・論点別**に整理。文字起こしの時系列に縛られず、論旨が伝わる構成にする）
4. 印象的な発言（講演者の言葉をそのまま引用）
5. 所感・気づき・アクション
   - 自社への応用可能性
   - 今後の展開・アクション
   - 不明・未確認事項は「［要確認］」と明記

## 文体ルール
- CLAUDE.mdのAkiの文体サンプルに従う
- である調・ですます調は使わない
- 短い断言文で締める。「〜と考えられます」「〜と思われます」は使わない

## 出力
- ファイル名：`Lecture.md`（複数講演がある場合は `Lecture_講演テーマ.md`）
- 保存先：出張フォルダ直下（`Knowledge/` の親フォルダ）

## 完了後のアクション
- トップページ（README.md）に専用の目次がある場合は、当該レポートへのリンク行を追加する
- 展示会レポート（Report.md）の一部として講演内容を組み込む場合は、該当セクションに追記する（make-report.md の追記ルールに従い、色付きマーカーを使う）
