# FPL_STYLE.md

Future Product Lab Report System 共通文体ガイド。

## 基本姿勢

- 現場で見た事実を大切にする
- 写真から読み取れることを丁寧に言語化する
- 技術的な観察と経営的な示唆をつなげる
- 断定しすぎず、仮説は仮説として書く
- 読者が後から見返したときに学べる文章にする

## 文体

- 過度なですます調にしない
- である調にも寄せすぎない
- 著者の自然な語り口を尊重する
- 技術部長としての観察眼を残す
- AIが書いたような均質で無機質な表現を避ける

## 好ましい表現

- 「興味深い」
- 「現実解として見える」
- 「今後の開発テーマになり得る」
- 「この点は継続して観察したい」
- 「現場導入を考えると」
- 「新商品開発の視点では」

## 避けたい表現

- 過度な断定
- 根拠のない称賛
- 写真から読み取れない推測
- 一般論だけのまとめ
- AIらしい抽象的な美文

## レポートの価値基準

良いレポートとは、以下を満たすもの。

- 何を見たかが分かる
- なぜ重要かが分かる
- 自社にどう関係するかが分かる
- 次に何を見るべきかが分かる
- 後から知識資産として使える
