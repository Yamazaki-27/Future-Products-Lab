# 展示会レポート作成

## 情報ソースの優先順位
1. Nippou.txt（最優先）
2. その他のMarkdownファイル
3. 写真のタイムスタンプと内容
4. Web検索（補足のみ・ソース必須）

## 作業方針
- Report.mdの作成・編集は、いちいち確認を求めずに進める
- トップページ（README.md）の編集も、確認を求めずに進める

## 不明な点の扱い
- 情報不足の場合、推測・創作で補わない
- 不明箇所は「［要確認：〇〇が不明］」と明記して空欄にする

## 写真・動画の前処理
- 動画ファイル（.MOV など）が含まれている場合は削除する
- 全ての写真（採用・unUsed 問わず）を横幅800ピクセル程度にリサイズする（sips コマンドで一括処理）
- ほぼ同じ構図で被写体・内容が全く同じ写真が複数ある場合のみ、最も良い1枚を選ぶ（少しでも内容・情報が異なれば両方採用する）
- 採用しなかった写真は、imagesフォルダー内に unUsed フォルダーを作成して移動させる

## 写真の使い方
- 全枚使用・省略しない
- 話の流れに沿って挿入
- 写真1〜2枚＋説明3行以内をセットで配置
- 写真の解説は必要・タイトルは不要
- 写真は `<img src="images/ファイル名" width="800">` で挿入し、解説文は写真の下に配置する

## 既存レポートへの追記
- Report.md がすでに存在する場合、新たに追加された写真を確認して、内容に沿った適切な位置に追記する

## 章立て
1. サマリー
2. 展示会概要・参加者
3. 視察の目的
4. 背景（環境変化）
5. 内容（タイムスタンプ順・時系列）
6. まとめ
   - 視察全体を通じた気づき・所感
   - 今後の展開・アクション
   - 不明・未確認事項は「［要確認］」と明記

## 文体ルール
- CLAUDE.mdのAkiの文体サンプルに従う
- である調・ですます調は使わない

## インターネット情報の使用ルール
- ソース（URL・サイト名・取得日）を本文中に明記

## 出力
- ファイル名：Report.md
- 保存先：該当サブフォルダーの直下

## 完了後のアクション
- トップページ（README.md）の「出張報告書」目次テーブルに、当該レポートへのリンク行を追加する
