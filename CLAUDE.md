# CLAUDE.md

## プロジェクト概要

2026年4月に実施した、山崎部長・橋本GM同行による米国・欧州出張の記録リポジトリ。
社内でのGit/GitHub普及活動の一環として、出張報告をGitHubで管理する取り組み。

## ファイル構成

| ファイル/フォルダ | 内容 |
|---|---|
| `README.md` | リポジトリの説明・Git操作ガイド・目次 |
| `archive_log.md` | 知識資産化の作業履歴 |
| `Reports/` | 展示会・視察レポート（202xxx-XXX/Report.md 形式） |
| `KnowledgeBase/Companies/` | 訪問・接触企業の情報ファイル |
| `KnowledgeBase/Ideas/` | 商品開発・提案アイデアファイル |
| `KnowledgeBase/Knowledge/` | 技術テーマ別の知識ベースファイル |
| `KnowledgeBase/Trends/` | 年別トレンドまとめ |
| `strategy/` | 中期商品開発戦略等のドキュメント |
| `kojinMD/` | 個人メモ・個別記録 |

## 運用ルール

- 各自、自分の名前でブランチを切って作業する
- Pull Request 経由でマージする（管理者：山崎部長が承認）
- Issues に感想・質問を投稿して個人が特定できるようにする

## 成果物（.mdファイル）の配置ルール

- スキル（make-report・make-lecture・dx-strategy等）が生成する成果物の.mdファイルは、実行しているフォルダー直下に置く
- 写真・文字起こし等の元データを格納するサブフォルダー（Images/・Knowledge/等）の中には置かない

## 画像ファイルの注意事項

- 写真はリサイズして容量を小さくしてから追加する
- 写真の採否は「20枚以下なら原則全て採用、20枚超は重複（同時間帯の食べ物・ホテルの部屋・同じ場所の集合写真・数秒差のほぼ同一ショット）をunUsedへ」（詳細は `.claude/commands/make-report.md` の「写真の使い方」を正とする）
- 写真やレポート内容の配置は原則として時系列を優先する（朝食 → 訪問 → 昼食 → ミーティング → 晩ごはん の順）
- 動画は外部リンク（YouTube・Googleドライブ等）で共有する
- 1ファイル100MB超はGitHubにpush不可


## PowerPoint作成ルール

### デザイン
- タイトル下のアクセントラインは使わない
- カラーバー・ストライプは使わない
- 毎スライド必ず画像か図を入れる
- レイアウトはスライドごとに変化させる
- 余白を十分に取り、すっきり見せる
- フォントはArial（本文）+ Cambria（見出し）

### 文章
- 1スライド3行以内
- 箇条書きより キーワード＋短い説明
- 「〜と考えられます」は使わない
- 短く断言する文体

### カラー
- ネイビー #1E2761 をベースに使う
- アクセントはアイスブルー #CADCFC
- クリーム・ベージュ系の背景は使わない


## Akiの文体サンプル

**詳細な文体規範は [Yamazaki-OSリポジトリの STYLE.md](../Yamazaki-OS/STYLE.md) を正とする**（5つのレジスターと8つのDNA、実文書からの引用付き。山崎個人の文体定義書としてYamazaki-OS側で一元管理）。以下はその要約サンプル。同じトーン・リズムで文章を書くこと。

※このリンクは同一マシン上の相対パス参照であり、本リポジトリをGitHub等で単体公開した場合はリンク切れになる（Yamazaki-OSは別リポジトリのため）。その場合は下記の要約ルールのみを参照すること。

---

会社の業績はこの20年間ずっと良い。
だが、それを自分たちの力だと、全員が勘違いしている。
新商品の売り上げ比率はわずか数%。
どちらかと言えば、生産性を低下させている。

返事や、笑顔や、些細な心遣いのある所作が、未来をつくる。
GitHubでも、AMRでも、AIでもなく。

---

### 文体の特徴（Claudeへの指示）
- 文末はである調（「〜だ」「〜である」「〜した」）
- ですます調は絶対に使わない
- 短い断言文で締める
- 口語と専門用語を自然に混在させる
- 結論を先に、理由は短く
- 「〜と考えられます」「〜と思われます」は使わない
- 現場感・体験の生々しさを残す
- トピックは「〜は、」で先に出してから内容を続ける（例：「訪問のきっかけは、〜、である。」。「〜がきっかけだ。」のようにトピックを述部に埋め込まない）
- 訪問先で聞いた話（伝聞情報）は、である／だ／というで断定せず、必ず末尾を裸の「とのこと。」で締める（「とのことである。」「とのことだ。」は冗長なので使わない）。実際に見た・撮影した内容は通常のである調でよい
- 体言止めを積極的に使う（「次の3点だ。」ではなく「次の3点。」）。名詞で終わる文をだ／であるで補わない
